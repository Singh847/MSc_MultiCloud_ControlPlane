Methodology

This project is an MSc dissertation prototype implementing an advanced enhanced multi-cloud governance control plane.

Architecture

  • Provider Clients: SDK / API clients per cloud (Azure real via Azure SDK; AWS/GCP mock or extendable to real).
  • Normalization Layer: converts heterogeneous provider resources into a unified Resource model.
  • Policy Engine: policy-as-code rules evaluate normalized resources and output findings.
  • Risk Scoring: calculates a risk score per resource based on exposure/encryption/privilege/cost.
  • Snapshot & Drift: snapshots capture state; drift compares two snapshots to detect changes over time.
  • Reporting: exports JSON + PDF; dashboard renders web views + charts.

Scoring Logic (High Level)

  • Public exposure increases risk
  • Unencrypted sensitive resources increase risk
  • Admin privileges / owner-like roles increase risk
  • High estimated costs adds operational risk (FinOps signal)

How Drift Works

Resources are indexed by resource_id. Between snapshots, the engine reports:

  • created: present in new but not old
  • deleted: present in old but not new
  • changed: same id but different fields
